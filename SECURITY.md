# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

- Do not create public GitHub issues for security vulnerabilities.
- Send the details to: ttg@traffic-tech.com
- Include:
  - Description of the issue
  - Steps to reproduce
  - Potential impact
  - Suggested fix (if available)

## Review Process

Once a report is received:

1. The security team will acknowledge the report within 3 business days.
2. The issue will be reviewed and validated.
3. A remediation plan and timeline will be prepared.
4. Security fixes will be tested before release.
5. The reporter may be contacted for additional clarification if required.


## Responsible Disclosure

We appreciate responsible disclosure and request that researchers:

- Avoid exploiting vulnerabilities beyond what is necessary for verification
- Do not access or modify user data
- Do not perform destructive testing
- Allow reasonable time for remediation before public disclosure
